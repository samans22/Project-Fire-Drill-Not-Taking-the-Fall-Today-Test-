## 三、优先级路线图（整合版）

### 🚨 P0 — 紧急（2-3 周）：主题联动 + 内容扩充

> 目标：解决"主题与内容两张皮"，事件池翻倍，角色出场均衡

| # | 改造项 | 工作量 | 影响 | 依赖 |
|---|--------|--------|------|------|
| **P0-1** | 主题驱动事件池 Level 1 | 3-5天 | 解决核心问题——主题不再是摆设 | themes.json + events.js |
| **P0-2** | 主题驱动事件池 Level 2 | 2-3天 | 同一事件不同主题不同文本 | 依赖 P0-1 |
| **P0-3** | 扩充专属事件到 40+ | 5-8天 | 内容翻倍，项目差异化 | 依赖 P0-1 |
| **P0-4** | 角色出场再平衡 | 3-5天 | 财务小周/甲方群/小许 从 1→5+ | 与 P0-3 并行 |

#### P0-1 详细方案：主题加权事件池

**核心算法**：
```
pickEvent(state):
  1. 优先：连锁事件（不变）
  2. 获取可用事件池
  3. 查当前主题的 eventPool.boosted 列表
  4. 70% 概率：从 boosted ∩ available 中抽取
  5. 30% 概率：从 available ∖ boosted 中抽取
  6. boosted 为空时：全部从 available 抽取
```

**themes.json 新增字段**：
```json
{
  "id": "client_adds_requirements",
  "text": "客户突然加需求",
  "modifier": { "time": -1, "satisfaction": -1 },
  "eventPool": {
    "boosted": ["E003","E011","E013","E023"],
    "boostWeight": 3
  }
}
```

#### P0-2 详细方案：主题文本覆写

**themes.json 新增字段**：
```json
{
  "textOverrides": {
    "E011": {
      "text": "周五下午4:47，甲方群的@所有人让你心头一紧——距离下班还有73分钟"
    }
  }
}
```

覆写在事件抽取后、UI 渲染前应用。不改变 effects，只改变叙事文本。

### 🟡 P1 — 高优先级（本月内 2-3 周）：玩法深度

> 目标：选择不再是终点，游戏有节奏感和紧张感

| **P1-4** | 主题驱动事件池 Level 3 | 3-5天 | 主题独家事件 | 依赖 P0-3 |

#### P1-1 详细方案：多阶段事件

选 5-8 个关键事件改造为 2-3 阶段：

```
示例："客户不满意初步方案"
  阶段1：收到客户抱怨 → 选择初步应对
  阶段2：客户反应（取决于阶段1）→ 再次选择
  阶段3：最终结果 → 综合反馈
```

覆盖四类职场场景：客户危机、技术故障、合规审查、团队冲突。

#### P1-2 详细方案：难度曲线

| 天数 | 难度 | 效果幅度 | 连锁概率 | 特殊机制 |
|------|------|----------|----------|----------|
| Day 1-3 | 简单 | ±1 为主 | 低 | 教学引导事件优先 |
| Day 4-6 | 中等 | ±2 为主 | 中 | 连锁事件开始出现 |
| Day 7-9 | 困难 | ±2~±3 | 高 | 高风险高回报选项 |
| Day 10 | 最终 | ±3 | 最高 | 最终审判日专属事件 |

### 新增数据通道

```
main.js:startNewGame()
  │  themeObj = themes[random]
  │  gameState.theme = themeObj.text
  │  gameState.themeId = themeObj.id        ← 新增
  │  gameState.themeEventPool = ...         ← 新增
  │
  ▼
events.js:pickEvent(state)
  │  读取 state.themeId                      ← 新增
  │  查 themes 对应 eventPool                ← 新增
  │  70/30 加权抽取                          ← 新增
  │  文本覆写                                ← 新增
  │
  ▼
ui.js:showEvent()
  │  渲染已覆写的文本                        ← 无改动
```

## 七、风险与注意事项

1. **内容生产是瓶颈**：P0-3 需要写 19+ 个新事件（每个 3 选项 + 反馈 + 提示），创意写作量最大
2. **主题加权不要过度**：70/30 分配如果 boosted 事件太少（如只有 2-3 个），会导致重复。每个主题至少配 4-5 个 boosted 事件
3. **先做 Level 1，验证后再做 Level 2+3**：避免一次性改动太大导致调试困难
4. **角色再平衡与新事件同步**：写新事件时注意 speaker 分配，自然达成再平衡
5. **每日配额改动影响较大**：P1-2 改变了游戏循环的底层逻辑，需要充分测试
